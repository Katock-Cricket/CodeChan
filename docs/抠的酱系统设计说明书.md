# 抠的酱（CodeChan）系统设计说明书

## 1. 项目概述

CodeChan 是一款以“二次元 3D 伴侣”为前端形态的 VS Code / Cursor 插件。系统将代码 Agent 的阅读、思考、写入、执行命令等行为转化为可观察的角色动作、表情、语音和 IDE 联动，让开发助手既能完成实际编程任务，也能提供陪伴式交互体验。

系统坚持以下设计边界：

- ASR 与 TTS 只使用本地模型，不调用云端音频服务。
- 云端服务仅用于 LLM 文本推理，包括闲聊、意图路由、情境回复和 Agent 指令整理。
- Agent 集成坚持使用定制化 OpenCode Fork，以保证事件质量和细粒度。
- 密钥只允许来自系统环境变量或 VS Code SecretStorage，不明文落盘。

## 2. 宏观架构

系统采用“VS Code 插件 + Webview 前端 + 本地 Python 后端 + OpenCode Fork + 云端 LLM”的混合架构。

```text
┌──────────────────────────────────────────────────────────────┐
│ VS Code / Cursor Extension Host                              │
│  ├─ Extension Shell                                           │
│  │   ├─ Commands / Keybindings / Sidebar / Settings           │
│  │   ├─ SecretStorage / Env Key Resolver                      │
│  │   └─ VS Code API Bridge                                    │
│  └─ Webview                                                   │
│      ├─ Vue 3 UI                                              │
│      └─ Avatar Runtime: Three.js + @pixiv/three-vrm           │
└───────────────────────────────▲──────────────────────────────┘
                                │ WebSocket + postMessage
┌───────────────────────────────▼──────────────────────────────┐
│ Local Python Server                                            │
│  ├─ Protocol Layer                                             │
│  ├─ Intent Router / Conversation Orchestrator                  │
│  ├─ ASR Adapter: Faster-Whisper                                │
│  ├─ TTS Adapter: IndexTTS-2                                    │
│  ├─ LLM Provider: OpenAI / Anthropic / DeepSeek compatible     │
│  └─ Agent Adapter: OpenCode Fork Process Manager               │
└───────────────────────────────▲──────────────────────────────┘
                                │ Hook RPC / Event Bus / stdin
┌───────────────────────────────▼──────────────────────────────┐
│ OpenCode Fork                                                  │
│  ├─ Native Terminal UI                                         │
│  ├─ Task Runtime                                               │
│  └─ CodeChan Hook: structured agent events                     │
└──────────────────────────────────────────────────────────────┘
```

## 3. 工程模块设计

### 3.1 VS Code Extension Shell

职责：

- 插件激活、关闭、重载和本地后端进程绑定。
- 注册 CodeChan 侧边栏、命令、设置页和 VS Code 内部快捷键。
- 管理 Webview 生命周期和前后端 `postMessage`。
- 调用 VS Code API 打开文档、滚动到范围、关闭临时文档。
- 通过 SecretStorage 保存 LLM API Key，通过环境变量读取外部密钥。

设计要点：

- 快捷键只使用 VS Code `contributes.keybindings`，不注册系统级全局热键。
- CodeChan 侧边栏提供聊天输入、Agent 指令输入、道具面板、连接状态和模型状态。
- OpenCode 原生终端输入保持独立，不属于 CodeChan 侧边栏，但其事件仍通过 Fork Hook 汇入可视化系统。

### 3.2 Avatar Runtime

职责：

- 加载 VRM 模型、贴图、材质、Humanoid 骨骼、Spring Bone 和表情。
- 维护 Three.js 渲染循环、摄像机、灯光、LookAt、鼠标射线检测。
- 维护动作栈、动画混合、表情混合和 Lip-sync。
- 处理 Webview 内角色点击、身体区域识别和道具命中。

技术选型：

- Webview UI：Vue 3 + Vite。
- 状态管理：Pinia。
- 3D 渲染：Three.js。
- VRM 支持：`@pixiv/three-vrm`。
- 动画播放：Three.js `AnimationMixer`，动作切换使用 `crossFadeTo()`。

### 3.3 Local Server

职责：

- 启动 FastAPI 服务，提供 WebSocket、健康检查和本地控制接口。
- 接收音频流并调用 ASR Adapter。
- 调用云端 LLM 完成意图路由、闲聊、情境回复和 Agent 指令整理。
- 调用 TTS Adapter 合成本地角色语音，并以二进制帧推给前端。
- 维护会话状态、Agent 当前上下文、任务队列、模型加载状态和错误状态。

技术选型：

- Python + FastAPI。
- WebSocket 使用 FastAPI 原生 WebSocket 或 `websockets`。
- 异步任务使用 `asyncio`，耗时推理可放入线程池或独立 worker。
- 本地日志使用标准 `logging` + 按日期滚动文件。

### 3.4 Agent Adapter

职责：

- 管理 OpenCode Fork 进程生命周期。
- 向 OpenCode Fork 注入来自 CodeChan 的任务。
- 接收 Fork Hook 输出的结构化事件。
- 标准化 Agent 状态并推送给 Local Server 和前端。
- 管理任务并发、取消、超时、异常恢复和日志归档。

OpenCode 集成采用 Fork 内部 Hook，不以 stdout 文本解析作为主路径。Fork 内部在事件总线或调度器层输出结构化事件，确保能拿到稳定、细粒度、可版本化的上下文。

### 3.5 Protocol Layer

职责：

- 定义 WebSocket 统一消息信封。
- 定义文本帧、音频二进制帧、错误码、能力协商和协议版本。
- 定义 Agent 事件、IDE 命令、动作命令、表情命令和音频流事件。

### 3.6 Config Layer

职责：

- 管理 persona、props、actions、model paths、LLM provider、路由策略和资源策略。
- 校验用户导入的 VRM、TTS、ASR 模型。
- 维护默认配置、用户配置和运行时临时状态的边界。

配置文件可保存非敏感字段；敏感字段仅保存 SecretStorage 引用或环境变量名。

## 4. 核心技术选型

| 子系统 | 首发选型 | 可替换策略 |
|--------|----------|------------|
| IDE 宿主 | VS Code Extension API | 兼容 Cursor 的 VS Code 扩展环境 |
| Webview UI | Vue 3 + Vite | 可替换为其他前端框架，但不建议首发变更 |
| 3D 渲染 | Three.js | Avatar Runtime 内部封装渲染接口 |
| VRM 解析 | `@pixiv/three-vrm` | 随 VRM 标准升级替换 |
| 本地后端 | Python + FastAPI | 后端接口通过 WebSocket 协议隔离 |
| ASR | Faster-Whisper | ASR Adapter 支持本地模型替换 |
| TTS | IndexTTS-2 | TTS Adapter 支持 VITS、Fish Speech 等本地模型 |
| LLM | OpenAI-compatible / Anthropic / DeepSeek | LLM Provider 接口统一 |
| Agent | OpenCode Fork | Agent Adapter 预留未来扩展，但首发坚持 Fork |

## 5. 意图路由设计

### 5.1 输入来源

系统输入分为以下来源：

- `ui_chat`：CodeChan 侧边栏聊天框。
- `ui_agent`：CodeChan 侧边栏 Agent 指令框。
- `voice`：用户语音经 ASR 后得到的文本。
- `avatar_interaction`：点击身体区域、道具使用等 Webview 显式事件。
- `opencode_terminal`：OpenCode 原生终端输入产生的 Agent 事件。

### 5.2 路由目标

| 路由目标 | 说明 |
|----------|------|
| `interaction` | 即时身体互动或道具互动，触发 P0 |
| `agent_task` | 编程任务，注入 OpenCode Fork |
| `action_only` | 只触发角色动作或表情 |
| `chat` | 闲聊回复，触发 LLM + TTS + P1 |
| `clarify` | 低置信度时追问用户 |

### 5.3 优先级规则

1. 显式 UI 事件优先级最高。道具、身体点击等事件直接进入 `interaction`。
2. CodeChan 侧边栏中显式选择 Agent 模式时，直接进入 `agent_task`。
3. CodeChan 侧边栏中显式选择 Chat 模式时，默认进入 `chat`，除非用户文本包含高风险操作并明确要求执行。
4. 语音或自由文本中出现“打开/修改/运行/分析/检查/生成/修复/提交”等代码任务意图时，进入 `agent_task`。
5. 文本包含摸头、转圈、笑一个、看这里等动作意图时，进入 `action_only` 或 `interaction`。
6. 同时包含 Agent 任务和闲聊时，优先创建 Agent 任务，并生成一条短确认回复。
7. 同时包含动作和闲聊时，先触发动作，再生成闲聊回复。
8. 置信度不足时进入 `clarify` 或 `chat`，不得误触发代码修改、命令执行或文件写入。

### 5.4 路由实现

首发采用“规则预判 + LLM 分类”的混合方案：

- 规则层处理显式来源、UI 模式、道具事件和明显命令关键词。
- LLM 分类层处理自由口语、混合意图和上下文相关表达。
- 路由结果必须包含 `route`、`confidence`、`reason`、`extracted_task`、`response_hint`。

```json
{
  "route": "agent_task",
  "confidence": 0.86,
  "reason": "用户要求检查 README 并总结项目",
  "extracted_task": "检查当前项目 README，并总结项目用途",
  "response_hint": "短确认即可，不要提前承诺结果"
}
```

## 6. 关键交互流程

### 6.1 语音闲聊

1. 用户点击麦克风按钮或触发 VS Code 内部快捷键开始录音。
2. Webview 使用 Web Audio API 捕获音频，发送 `audio_start`、二进制音频帧、`audio_end`。
3. Local Server 调用本地 ASR Adapter 得到文本。
4. Intent Router 将文本路由到 `chat`。
5. LLM Provider 生成符合角色人设的回复、情绪标签和动作建议。
6. TTS Adapter 使用本地 IndexTTS-2 合成语音。
7. 后端推送 `text_response`、`expression_command`、`action_command`、`audio_start`、音频二进制帧和 `audio_end`。
8. Avatar Runtime 播放 P1 动作、表情和 Lip-sync。

### 6.2 语音下发 Agent 任务

1. 用户通过语音说出“帮我看看项目的 README”。
2. ASR 产出文本，Intent Router 路由到 `agent_task`。
3. 若当前没有活跃 Agent 任务，Agent Adapter 将任务注入 OpenCode Fork。
4. 后端可先生成一条短确认回复，如“收到，抠的酱这就去看 README。”
5. OpenCode Fork 运行任务并通过 Hook 产生结构化事件。
6. 后端将事件映射为 `agent_status`、`ide_command`、`action_command`。
7. 前端打开相关文档、滚动到范围，角色进入 P2 任务动画。
8. Agent 完成后恢复 P3 待机，并按临时文档规则关闭可视化临时打开的文档。

### 6.3 OpenCode 原生终端任务可视化

1. 用户在 OpenCode 原生终端中直接输入任务。
2. OpenCode Fork 正常执行，终端 UI 保持原有体验。
3. Fork Hook 并行输出结构化事件到 Local Server。
4. CodeChan 不接管终端输入，但消费事件用于角色动作、IDE 打开文档和状态展示。

### 6.4 复杂互动与 P0/P1 队列

1. 用户选择道具并点击角色，或用言语进行嘉奖/惩罚。
2. Webview 立即触发 P0 动作和预置语音，通常持续 3-5 秒。
3. Local Server 同时读取当前 Agent 上下文，调用 LLM 生成情境化回复。
4. TTS Adapter 在后台合成 P1 语音。
5. 若 P1 早于 P0 完成，进入 `pending_after_p0` 队列等待。
6. P0 结束后，动作栈调度器播放 P1 对话动作和语音。
7. P1 结束后恢复 P2 或 P3。

## 7. Agent 进程管理设计

### 7.1 生命周期状态

```text
not_started -> starting -> ready -> busy -> waiting_user -> stopping
       ^          |          |       |          |             |
       |          v          v       v          v             v
       +------ failed <--- crashed <- timeout <- cancelled <- stopped
```

| 状态 | 含义 |
|------|------|
| `not_started` | 尚未启动 OpenCode Fork |
| `starting` | 正在启动进程和建立 Hook 通道 |
| `ready` | 进程健康，可接收任务 |
| `busy` | 正在执行 Agent 任务 |
| `waiting_user` | Agent 请求用户确认或输入 |
| `stopping` | 正在停止进程 |
| `stopped` | 已正常停止 |
| `cancelled` | 当前任务被用户取消 |
| `timeout` | 启动、心跳或任务超时 |
| `crashed` | 进程异常退出 |
| `failed` | 启动失败、Hook 失败或不可恢复错误 |

### 7.2 启动流程

1. Extension Shell 启动 Local Server。
2. Local Server 根据工作区路径和用户配置创建 Agent 会话。
3. Agent Adapter 启动 OpenCode Fork 子进程，记录 PID、cwd、启动参数和日志路径。
4. OpenCode Fork 初始化 Hook 通道，发送 `agent_hello`。
5. Agent Adapter 校验协议版本和能力列表。
6. 状态进入 `ready`，前端显示 Agent 可用。

启动超时默认 30 秒，可配置。启动失败时展示错误原因和日志入口。

### 7.3 健康检查与心跳

- OpenCode Fork 每 5 秒发送心跳。
- Agent Adapter 记录 `last_heartbeat_at`。
- 连续 3 次心跳缺失视为 `timeout` 或 `crashed` 候选状态。
- 若进程仍存在但 Hook 无响应，先尝试重建 Hook 通道；失败后提示用户重启 Agent。

### 7.4 终止与重启

- 正常停止先发送 graceful stop 请求。
- 若 5 秒内无响应，发送子进程终止信号。
- 若仍无法退出，再提示用户手动处理，不在文档层面要求强制杀死所有子进程树。
- 重启时需清理当前任务状态、临时文档标记、P2 动作和 Hook 会话。

### 7.5 异常恢复

| 异常 | 处理策略 |
|------|----------|
| Agent 启动失败 | 展示失败原因、日志路径、重试按钮 |
| Hook 协议不兼容 | 阻止可视化联动，提示 Fork 版本不匹配 |
| Agent 运行中崩溃 | 标记 `crashed`，停止 P2，保留日志，允许一键重启 |
| 任务超时 | 标记当前任务 `timeout`，询问用户取消、继续等待或重启 |
| LLM 超时 | 使用短错误回复，不影响 Agent 进程 |
| TTS 失败 | 降级为文本显示和无声动作，不调用云端 TTS |
| ASR 失败 | 提示用户重试或改用文本输入 |

### 7.6 日志设计

日志分层：

- Extension 日志：插件激活、Webview 生命周期、IDE 命令执行。
- Webview 日志：渲染初始化、模型加载、动作调度错误。
- Local Server 日志：WebSocket、路由、LLM、ASR/TTS、任务状态。
- Agent 日志：OpenCode Fork stdout/stderr、Hook 事件、任务注入。

日志要求：

- 默认 INFO，调试模式可开启 DEBUG。
- API Key、Authorization Header、SecretStorage 值必须脱敏。
- 音频二进制内容不写入日志。
- Agent 事件可记录结构化摘要，但大段文件内容默认不落盘。

### 7.7 超时策略

| 场景 | 默认超时 | 处理 |
|------|----------|------|
| Local Server 启动 | 20 秒 | 提示后端启动失败 |
| OpenCode Fork 启动 | 30 秒 | 提示 Agent 启动失败 |
| Hook 握手 | 10 秒 | 标记协议通道失败 |
| ASR 单次识别 | 30 秒 | 提示识别失败 |
| LLM 首包 | 60 秒 | 提示模型响应慢，可继续等待 |
| TTS 首包 | 45 秒 | 降级文本与动作 |
| Agent 无事件心跳 | 15 秒 | 标记心跳异常，尝试恢复 |

这些值为默认配置，可在高级设置中调整。

### 7.8 并发任务策略

首发版本同一工作区只允许一个活跃 Agent 任务。

- 若 Agent `ready`，新任务直接启动。
- 若 Agent `busy`，来自 CodeChan 的新任务进入待确认状态，用户可选择排队、取消当前任务后启动、或放弃新任务。
- 若任务来自 OpenCode 原生终端，CodeChan 不阻止，但会将当前可视化状态切换到该终端任务。
- 取消当前任务必须有明确用户确认。
- 任务队列首发最多保留 5 条，超过后提示用户清理。

## 8. IDE UI 联动设计

### 8.1 文档打开与滚动

Agent 事件提供文件路径时，Extension Shell 调用 VS Code API 打开文档：

```ts
const document = await vscode.workspace.openTextDocument(uri);
const editor = await vscode.window.showTextDocument(document, { preview: true });
```

若事件包含范围：

```ts
editor.revealRange(range, vscode.TextEditorRevealType.InCenterIfOutsideViewport);
```

系统不做高亮，不做额外 Diff 或预览渲染。

### 8.2 临时文档关闭策略

当 CodeChan 为可视化自动打开文档时，记录：

```ts
{
  uri: "file:///workspace/README.md",
  opened_by_codechan: true,
  opened_at: 1700000000000,
  event_id: "agent-event-id",
  user_touched: false
}
```

如果用户出现以下行为，则不自动关闭：

- 在该文档中编辑内容。
- 主动切换到该文档并停留超过配置阈值。
- 将该标签固定或从 preview 变为正式标签。
- 通过命令选择“保留当前文档”。

Agent 对该文件的可视化事件结束后，若 `user_touched = false`，Extension 可关闭该临时文档。关闭前需避免影响用户当前正在编辑的活动文档。

## 9. WebSocket 通信协议

### 9.1 消息信封

所有非音频消息均为 UTF-8 JSON 字符串帧：

```json
{
  "version": "1.0",
  "type": "string",
  "msg_id": "uuid4",
  "session_id": "uuid4",
  "timestamp": 1700000000000,
  "payload": {}
}
```

音频数据使用二进制帧单独传输，并通过相邻的 `audio_start` / `audio_end` 与 `session_id` 关联。

### 9.2 前端到后端消息

| type | 触发场景 | payload |
|------|----------|---------|
| `client_hello` | WebSocket 建立 | `client_version`, `capabilities` |
| `audio_start` | 开始录音 | `audio_session_id`, `format`, `sample_rate` |
| `audio_chunk` | 录音中 | 二进制帧 |
| `audio_end` | 结束录音 | `audio_session_id` |
| `text_input` | 文字输入 | `text`, `mode: "chat"|"agent"|"auto"` |
| `interaction_event` | 点击或道具 | `sub_type`, `region?`, `prop_id?` |
| `agent_task_control` | 任务控制 | `action: "cancel"|"queue"|"replace"`, `task_id?` |
| `settings_update` | 设置保存 | `section`, `data` |
| `model_check` | 模型校验 | `model_type`, `path`, `adapter` |

### 9.3 后端到前端消息

| type | 触发场景 | payload |
|------|----------|---------|
| `connection_ack` | 握手成功 | `server_version`, `protocol_version`, `capabilities` |
| `route_result` | 意图路由完成 | `route`, `confidence`, `reason` |
| `text_response` | 文本展示 | `content`, `role`, `streaming?` |
| `action_command` | 动作控制 | `action_id`, `priority`, `duration_ms`, `queue_policy` |
| `expression_command` | 表情控制 | `expression_id`, `intensity`, `blend_ms` |
| `audio_start` | TTS 开始 | `audio_session_id`, `format`, `sample_rate` |
| `audio_chunk` | TTS 音频 | 二进制帧 |
| `audio_end` | TTS 结束 | `audio_session_id` |
| `lipsync_frame` | 口型帧 | `phoneme?`, `viseme?`, `intensity`, `offset_ms` |
| `agent_status` | Agent 状态 | `state`, `task_id`, `context`, `action_id?` |
| `ide_command` | IDE 控制 | `command`, `file?`, `range?`, `temporary?` |
| `system_status` | 系统状态 | `backend`, `agent`, `asr`, `tts`, `llm` |
| `error` | 错误 | `code`, `message`, `recoverable`, `detail?` |

### 9.4 IDE 命令类型

| command | 说明 |
|---------|------|
| `open_file` | 打开文档 |
| `reveal_range` | 滚动到范围 |

## 10. 动作栈调度设计

### 10.1 数据结构

```ts
type ActionPriority = "P0" | "P1" | "P2" | "P3";

interface AvatarAction {
  action_id: string;
  priority: ActionPriority;
  type: "event" | "state" | "continuous";
  duration_ms: number;
  started_at?: number;
  loop: boolean;
  queue_policy?: "interrupt" | "replace_same_priority" | "after_p0";
  audio_session_id?: string;
}
```

动作栈：

```ts
const actionStack: Record<ActionPriority, AvatarAction | null> = {
  P0: null,
  P1: null,
  P2: null,
  P3: idleAction
};
```

### 10.2 调度规则

1. P0 立即打断 P1/P2/P3。
2. P1 可打断 P2/P3，但不能打断正在播放的 P0。
3. P2 在 Agent 状态持续期间循环，Agent idle 后清空。
4. P3 永远存在，作为兜底。
5. 同优先级新动作替换旧动作。
6. 若 P1 在 P0 播放期间准备完成，进入 `pending_after_p0`，待 P0 完成后播放。
7. 语音和动作绑定时，以音频 session 的播放结束作为 P1 完成信号；若无音频，则使用 `duration_ms` 或动画自然时长。

### 10.3 复杂互动队列

```text
prop_use -> P0(action + preset audio, 3-5s)
         -> background LLM + local TTS
         -> pending_after_p0
         -> P1(contextual reply + lipsync)
         -> restore P2/P3
```

## 13. LLM 与密钥设计

### 13.1 Provider 配置

```json
{
  "llm": {
    "provider": "deepseek",
    "base_url": "https://api.deepseek.com",
    "model": "deepseek-chat",
    "api_key_source": "secret_storage",
    "api_key_ref": "codechan.llm.deepseek"
  }
}
```

`api_key_source` 可选：

- `secret_storage`：从 VS Code SecretStorage 读取。
- `environment`：从系统环境变量读取。

配置文件不保存明文 API Key。

### 13.2 上下文最小化

发送给云端 LLM 的上下文按用途分级：

- 闲聊：角色人设 + 用户输入。
- 意图路由：用户输入 + 当前 UI 模式 + 简短 Agent 状态。
- 复杂互动：用户互动事件 + 当前 Agent 状态摘要。
- Agent 指令整理：用户任务 + 当前工作区摘要或用户允许的上下文。

默认不上传完整语音、音频二进制和未必要的完整文件内容。

## 14. 性能与降级设计

### 14.1 渲染性能

- 默认目标为 30 FPS 以上，轻负载场景力争接近 60 FPS。
- 提供渲染质量选项：阴影、抗锯齿、Spring Bone、后处理、动画更新频率。
- Webview 隐藏或失焦时可降低渲染频率。

### 14.2 本地模型资源

- ASR/TTS 支持懒加载。
- 长时间空闲可释放模型或保持热加载，由用户配置决定。
- 设备选择支持 `auto`、`cpu`、`cuda`、`mps`。

### 14.3 降级策略

系统不降级到云端音频服务。

| 失败点 | 降级结果 |
|--------|----------|
| ASR 不可用 | 关闭语音输入，保留文本输入 |
| TTS 不可用 | 静音显示文本，保留表情和动作 |
| VRM 加载失败 | 使用默认模型或占位模型 |
| LLM 不可用 | 保留本地预置互动，Agent 任务需提示 LLM 不可用 |
| Agent 不可用 | 保留闲聊和角色互动，禁用 Agent 任务入口 |

降级必须在 UI 中可见，不静默隐藏。

## 16. 开工优先级建议

首发实现建议按以下顺序推进：

1. Extension Shell + Webview 基础通信。
2. Avatar Runtime 最小闭环：默认 VRM、P3 待机、P1 说话、基础 Lip-sync。
3. Local Server WebSocket + LLM Provider + SecretStorage / 环境变量密钥。
4. 本地 TTS IndexTTS-2 与音频下发。
5. 本地 ASR Faster-Whisper 与语音输入。
6. OpenCode Fork Agent Adapter：启动、任务注入、Hook 事件、状态可视化。
7. IDE 打开文档与滚动范围、临时文档关闭策略。
8. 道具面板、P0/P1 队列、复杂互动上下文回复。
9. 模型导入、配置校验、日志诊断和降级体验。
